## 2. Screen Designs

### 2.1 Beranda (Home Hub)

**Pattern:** Gojek-style compact hub. Everything fits in ~1.5 screens max.
**No emojis.** Use Ionicons only.

#### Sections (top to bottom):

1. **Hero Wallet Card** (dark slate `#0F172A`)
   - Greeting: "Halo, {firstName}"
   - Balance: large number + "Poin" suffix
   - Subtitle: "Setara Rp{amount} sembako"
   - No progress bar. Clean.

2. **Quick Action Grid** (2x2 icon grid)
   - Each cell: icon circle (colored bg) + label (2 words max)
   - Actions:
     - **Setor Sampah** â€” icon: `basket-outline`, bg: `#ECFDF5`, â†’ navigate to Setor tab with `type=trash`

     - **Bantuan** â€” icon: `call-outline`, bg: `#F1F5F9`, â†’ WhatsApp deep link

3. **Status Terakhir** (latest report status)
   - Compact card showing most recent active report
   - Content: trash type, address (truncated), **explicit status chip** (not timeline dots)
   - Status chips: "Menunggu" (amber), "Dijadwalkan" (blue), "Diambil" (green), "Selesai" (green filled)

   - Tap â†’ navigate to Riwayat tab

4. **Reward Terdekat** (nearest redeemable reward)
   - Single compact card showing the cheapest reward the user can almost afford
   - Content: reward name, points needed, small progress indicator
   - Example: "Minyak Goreng 1L â€” Kurang 2.750 poin lagi"
   - If user can afford something: "Anda bisa menukar: Minyak Goreng 1L"

   - "Butuh bantuan? Hubungi petugas desa"

6. **Dampak Desa** (community impact â€” compact inline)
   - Single row: "{kg} terkumpul Â· {warga} aktif Â· {titik} dibersihkan"
   - Subtle, not a big card. Just a footer-style info strip.

7. **Sync Status Footer**
   - Offline/online indicator text (existing behavior, restyled)

#### What's REMOVED from current Home:

## 3. Design System Changes

### Color Usage Rules
- **Green (`#2D6A4F`)**: ONLY for primary CTAs, success states, active indicators
- **Slate surfaces**: All backgrounds, cards, borders
- **Amber (`#D97706`)**: Processing/pending states
- **Blue (`#3B82F6`)**: Info states, points display
- **Red (`#EF4444`)**: Destructive actions only

### Typography Rules
- No emoji anywhere in the app
- Ionicons replace all emoji references
- Compact labels (2 words max on action grid)
- Status uses explicit chips, not abstract dot timelines

### Component Changes

| Component | Action |
|-----------|--------|
| `ResikStatusTimeline` | Keep â€” but ONLY used on Report Detail page |
| `ResikStatusChip` | Keep â€” used on Riwayat cards and Status Terakhir |

| `ResikCard` | Keep â€” update default radius/shadow |
| `ResikBadge` | Keep |
